## 2026-06-20 (env-example synced from master + GH org-level secrets locked)

- **Two-track env-and-secrets management — locked.** User direction (verbatim): *"write the all teh env example files in the .env.example also so everything is managed from one project and we set the github action secret on org chirag127 level."* Two locks land in the same conversation, captured as a single decision [`decisions/security/env-and-secrets-single-source.md`](./decisions/security/env-and-secrets-single-source.md). **Track A** (PUBLIC, key names only): canonical `.env.example` lives ONLY at master [`templates/.env.example`](../templates/.env.example); every other repo's `.env.example` is a verbatim copy synced via [`scripts/sync-env-example.sh`](../scripts/sync-env-example.sh). [Doppler](./services/secrets/doppler.md) stays upstream source of truth per [`decisions/security/secrets-management-doppler.md`](./decisions/security/secrets-management-doppler.md); org-level GH secrets are the runtime CI mirror. Repo-narrow keys (Chrome Web Store, AMO, Edge Partner, VS Code Marketplace, Open VSX publishers) use `--visibility selected --repos <comma-list>`. Helper script [`scripts/set-org-secrets-from-doppler.sh`](../scripts/set-org-secrets-from-doppler.sh) reads every key from `templates/.env.example`, fetches the value from Doppler, sets it at org level — idempotent, supports `--dry-run` and `--only KEY`.
