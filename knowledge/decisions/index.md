## How to add a new decision file

1. Pick a topic subdir (`architecture/`, `branding/`, `content/`, `infrastructure/`, `monetisation/`, `process/`, `security/`, `tooling/`).
2. Pick a kebab-case filename naming the concept clearly.
3. Use the OKF frontmatter shape from [`../_okf.md`](../_okf.md) — `type: decision`, `title`, `description`, `tags`, `timestamp`, `format_version`, `status`, optional `related:`.
4. Body has 4 sections: **Decision** (one sentence), **Why** (2–3 sentences), **Implications** (bullets), **Cross-refs** (markdown links).
5. Append a one-line entry at the top of this file under today's date.
6. Append a line to [`../log.md`](../log.md) with the date + path.
